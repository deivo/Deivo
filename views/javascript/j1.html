【JavaScript】变量提升、闭包、作用域链

vuePress-theme-reco Deivo    2019 - 2021

【JavaScript】变量提升、闭包、作用域链

Deivo 8/21/2018 JavaScript

# 变量提升

# 简介：

  • JS 代码自上而下执行之前，浏览器会提前处理一些事情（可以理解为：词法解析的一个环节，因为词法解析一般发生在代码的执行之前）

# 具体做来那些事情？

  1. 把当前上下文中所有带var提前声明，带function 关键字的进行提前的声明和定义
  2. 基于var或function在全局上下文中声明的变量，无论是严格模式还是非严格模式，都会映射到 GO（全局对象）一份，作为全局对象的属性；如果一个地方修改，另一个也跟着会修改。

在使用 es6 提供的 let/const 声明的变量，不存在变量提升和映射全局的特性，

# 注意 1：

  • 不论条件是否成立，都要进行变量提升（条件中带[[function]]在新版本浏览器中只会提前声明，不会在提前赋值了）
console.log(a, func)
if (!('a' in window)) {
  var a = 1
  function func () {}
}
console.log(a)
  • 上面代码，在全局上下文中，新版本浏览器输出结果：undefined undefined undefined，老版本浏览器（< ie10版本）输出结果：undefined 函数 undefined

# 注意 2：

  • 新版本的浏览器

    • 向下兼容 ES3/5 规范

      • 判断体和函数体等不在块级上下文，上下文中只有全局和私有
      • 不论条件是否成立，带 function 的都要声明 + 定义

    • 向上兼容 ES6 规范

      • 存在块级作用域：大括号中出现 let/const/function...都会认为是块级作用域 - 不论条件是否成立，带function的只提前声明，不会提前赋值

# 示例一

var a = 0
if (true) {
  a = 1
  function a () {}
  a = 21
  console.log(a)
}
console.log(a)
  • 上面代码执行，浏览器要兼容 ES3/5，向上兼容 ES6，function a在全局下声明过，也在私有下处理过，遇到此行代码，私有下不会在处理，但是浏览器会把当前代码“之前”(包括变量提升定义也要映射到全局)，所有对 a 的操作，映射给全局一份，以此兼容 ES3，但是它后面 的代码和全局没有任何关系

# 示例二

console.log(foo) //
{
  console.log(foo) //
  function foo () {}
  foo = 1
  console.log(foo) //
}
console.log(foo) //
  • 只有在除函数以外大括号中的函数“，在不同的浏览器中表现不一致：低版本是声明+定义,高版本只声明（高版本会把大括号当做块作用域）

# 函数arguments和行参之间关系

  • 条件：
    • 非严格模式下，arguments和形参存在映射机制
    • 严格模式下，arguments和形参映射机制不存在
function fn (x, y, z) {
  /**
   * 非严格模式下，映射机制
   *  x ---------- argument[0]
   *  y ---------- argument[1]
   *  z ---------- argument[2]
   */
  // arguments必须是传了值才会有值，如果没有传，是没有值
  x = 100
  console.log(x)
  arguments[1] = 200
  console.log(y)
  z = 300
  console.log(arguments[2])
}
fn(1, 2)
  • 以上示例，在非严格模式和严格模式下打印不同的返回结果

# 小结

  • 真实项目通常使用函数表达式来创建函数，因为这样在变量提升阶段只会声明函数，不会赋值
  • 函数表达式匿名函数具名化（虽然是起了名字，但是这个不能在外面访问）,但函数执行，在形成私有上下文中，会把这个具名化的名字作为私有上下文中的变量（值就是这个函数）来进行处理。
var fn = function AAA () {
  console.log('ok')
  console.log(AAA)
  console.dir(arguments.callee)
  // AAA(); 递归调用  而不用严格模式下都不支持的 arguments.callee了
}
fn()

# 闭包

# 概念

  • 函数执行会形成全新的私有上下文，这个上下文可能被释放，也可能不被释放，不论是否被释放，它的作用是：
    • 保护：划分一个独立的代码执行区域，再这个区域中有自己私有变量存储的空间，而用到的私有变量和其它区域中的变量不会有任何的冲突（防止全局污染）
    • 保存：如果上下文不销毁，那么存储的私有变量值也不会被销毁，可以被其下级上下文中调取使用
  • 函数执行，形成私有上下文，来保护和保存私有变量的机制，称之为 "闭包"，所以闭包是一种机制。

# 市面上认为的两种闭包：

  1. 函数执行形成的私有上下文不被销毁，才算 闭包
  2. 函数执行形成的私有上下文不被销毁，只有下级上下文用到了此上下文中的东西才算 闭包

# 不销毁的上下文

  • 函数执行形成私有的上下文，当前上下文进栈执行，经过一些系列操作（代码执行之前和代码执行），正常情况下，代码执行完，私有上下文会出栈（出站后被释放，以此节约栈内存的空间）；但是有特殊情况，如果当前上下文中的某个东西（一般是堆）被上下文以外的事物占用了，则当前上下文不会再出栈释放，也就是形成不销毁的上下文。

# 作用域链

# 概念：

  • 私有执行上下文中代码执行的时候，遇到一个变量，先看在当前作用域中是否存在，如果不存在，则向它的上级作用域中查找，如果该变量不存在，将再次查找直到找到全局，如果找到，则可以使用它，否则会报错，把这种查找的机制叫作用域链
