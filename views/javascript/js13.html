【JavaScript】JS中的数据类型

Deivo 11/1/2017 JavaScript

# js 中的数据类型

# 基本数据类型 （7 个）

  • 在浏览器的代码执行的底层机制中，基本数据类型就是值类型，是按照值来操作的。
    • number
    • string
    • boolean
    • null 表示空指针指向
    • undefined
    • symbol (es6 新增)
    • bigint (es6 新增)

# 引用数据类型 （2 个）

  • 在浏览器代码执行的底层机制中，引用数据类型操作的时候，引用的是堆内存的地址。
  1. object
  • 普通对象
  • 数组对象
  • 正则对象
  • 日期对象
  • Math 数学函数对象
  • ...
  1. function
  • 注意：为什么说 function 是引用数据类型了，是因为在面向对象中ptoto 原型链指向 Function 的 prototype

# 数字类型（number）

# 1. 有效数字： （例如：1，0，-1，-1.2）

let a = 1
let b = 0
let c = -1
let d = -1.2

# 2. NaN ：虽然不是有效的数字，但是它属于 number 数据类型

  • NaN 的特点 1： typeof 检测 NaN 返回值为 ”number“
  • NaN 的特点 2： NaN 和谁都不相等，包括 NaN 本身，返回值就是 false
  • NaN 的特点 3： NaN 和任何数学运算，都返回 NaN

# 3. Infinity：无穷大的值，它属于 number 类型

typeof Infinity //  "number"

# 4. -Infinity：负无穷大的值，它也属于 number 类型

typeof -Infinity //  "number"

# 检测有效数字的方法: isNaN()

# 1. 检测规则

  • 是有效数字返回：false，不是有效数字：返回：true。

# 2. 检测特点

  • 属于隐式转换（先进行 Number 隐式转换，在判断是否为有效数字）
typeof isNaN() //'boolean'
Number.isNaN() //false

isNaN() // true
isNaN(NaN) // true
isNaN(undefined) // true Number(undefined)=> NaN
isNaN({}) // true Number({})=> NaN

isNaN(true) // false
isNaN(null) // false
isNaN(37) // false

// strings
isNaN('37') // false: 可以被转换成数值37
isNaN('37.37') // false: 可以被转换成数值37.37
isNaN('37,5') // true
isNaN('123ABC') // true:  parseInt("123ABC")的结果是 123, 但是Number("123ABC")结果是 NaN
isNaN('') // false: 空字符串被转换成0
isNaN(' ') // false: 包含空格的字符串被转换成0

// dates
isNaN(new Date()) // false
isNaN(new Date().toString()) // true

isNaN('blabla') // true: "blabla"不能转换成数值
// 转换成数值失败， 返回NaN

# 3. 实现原理

var isNaN = function (value) {
  var n = Number(value)
  return n !== n
}
