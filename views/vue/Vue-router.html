# vue-router的原理及使用：

**原理：**当我们通过Vue.use注册vue-router路由的时候，会为用户提供两个全局组件 router-link和router-view，提供两个组件主要是路由有可能是 hash模式 或 history模式 ，通过两个组件来兼容，记下来就是根据不同的路径，渲染不同的组件。

使用：

  1. 导入vue-router路由

    import VueRouter from 'vue-router'

  2. 注册vue-router

    Vue.use(VueRouter);

  3. 添加路由配置，并且导出路由的实例

    export default new VueRouter({
    mode: 'hash',
    routes: [
        {
            path: '/',
            component: Home
        },
        {
            path: '/about',
            component: About,
            children: [
                {
                    path: 'a',
                    component: {
                        render: (h) => <h1>about a</h1>
                    }
                },
                {
                    path: 'b',
                    component: {
                        render: (h) => <h1>about b</h1>
                    }
                }
            ]
        }
    ]
})

  4. 在vue创建实例的时候，需要被注册到vue的实例中

    // main.js

new Vue({
  el: '#app',
  router, //注入router实例
  render:(h)=>h(App)
})

  5. 在vue的组件中可以通过两个属性分别获取对应的属性或方法

    this.$route  存放的是当前路由的属性等
this.$router 存放的当前实例中的方法等

# vue-router路由实现原理：

  1. 默认vue-router插件导出的是一个类**(VueRouter)**，用户会new来创建vur-router的实例

  2. 导出VueRouter身上有一个方法install

  3. 当用户Vue.use来注册VueRouter这个类的时候，会默认执行VueRouter这个类自身的install方法

  4. install方法默认接受两个参数，分别是：Vue 和options，为什么要传入Vue这个类呢，传入的这个Vue是Vse.use的时候和用户的版本保持一致，这样处理会兼容不同版本的Vue，如果默认导入Vue的时候，如果用户的Vue版本和install中Vue版本不一致，可能就会出现问题

  5. 当调用install方法的时候，会当前Vue挂载两个全局组件，给Vue.prototype创建两个属性$route和$router**

  6. install方法执行的时候，通过Vue.mixin放给所有的组件实例混入一个属性 router ，具体实现原理：通过Vue的全局钩子函数beforeCreate方法来实现，内部通过先判断父组件 this.$options.router来判断给父组件添加属性_routerRoot ，同时让_routerRoot等于当前的组件的Vue实例；给所有的子组件通过this.$parent来找到自己的父组件上的_routerRoot，这样机会让所有组件都能通过this._rouerRoot找到路由的router实例。总结：vue-router中install方法就是实现所有组件中路由实例的共享属性和状态

  7. install中需要调用vue-router的init方法，实现初始化操作，在根组件中初始化操作，并且把当前根组件的 “this” 传入

  8. VueRouter的核心：

    • 内部通过传入的options，对用户传入的options，调用createMatcher进行扁平化操作，createMatcher方法会返回两个重要的方法：match 和 addRoutes;

      1. match方法：通过路由来匹配组件
2. addRoutes方法：动态添加匹配规则

    • 映射表拼接完成之后，就需要根据映射表匹配不同的路径进行跳转
